## 1. Контекст

Данная реализация является базовым (эталонным) последовательным алгоритмом для умножения
двух разреженных матриц, представленных в формате CCS (Compressed Column Storage).
Матрицы содержат комплексные числа (`std::complex<double>`). Цель данной работы —
создать надежный, математически корректный фундамент для последующего распараллеливания
и проверки корректности многопоточных версий.

## 2. Постановка задачи

- **Вход:** Две матрицы `A` (размера $M \times K$) и `B` (размера $K \times N$) в формате CCS.
- **Выход:** Матрица `C = A \times B` (размера $M \times N$) в формате CCS.
- **Ограничения:** Умножение возможно только при условии совместимости размерностей:
  число столбцов левой матрицы (`A.count_cols`) должно быть равно числу строк правой
  матрицы (`B.count_rows`). Эта проверка выполняется на этапе `ValidationImpl`.
- **Алгоритм:** Вычисление линейной комбинации столбцов с использованием временного
  плотного вектора-аккумулятора. Значения, чей модуль меньше заданного порога `1e-15`,
  отбрасываются (считаются нулем).

## 3. Описание формата CCS

Формат CCS (Compressed Column Storage) хранит матрицу в виде трех массивов:

1. `values` — значения всех ненулевых элементов (сверху вниз, слева направо).
2. `row_index` — индексы строк, соответствующих элементам из `values`.
3. `col_index` — массив указателей, где `col_index[j]` указывает на индекс начала j-го
   столбца в массивах `values` и `row_index`. Размер этого массива равен `count_cols + 1`.

Такой формат делает крайне эффективным извлечение целых столбцов матрицы за $O(1)$,
но делает поиск конкретного элемента по строке долгим. Это напрямую диктует выбор
алгоритма умножения.

## 4. Схема алгоритма

Для умножения $C = A \times B$ в формате CCS используется подход "столбец на столбец".
Столбец $j$ матрицы $C$ является линейной комбинацией столбцов матрицы $A$,
где коэффициентами выступают элементы $j$-го столбца матрицы $B$.

**Алгоритм:**

1. Выделяется временный "плотный" вектор `dense_col` размером $M$ (число строк матрицы A)
   и массив логических маркеров `is_active`.
2. Внешний цикл проходит по всем столбцам матрицы `B` (от $j = 0$ до $N-1$).
3. Внутренний цикл извлекает все ненулевые элементы из $j$-го столбца `B`.
   Пусть это элемент $B_{k,j}$.
4. Из матрицы `A` извлекается столбец с индексом $k$.
5. Все элементы столбца $A_k$ умножаются на скаляр $B_{k,j}$ и прибавляются
   к соответствующим позициям в `dense_col`.
6. По завершении обработки столбца $B_j$, все затронутые элементы из `dense_col`
   (их индексы хранятся в `active_rows`) сортируются по возрастанию индекса строки.
7. Элементы, превышающие порог нуля, добавляются в итоговую матрицу `C`
   (в массивы `values` и `row_index`), после чего `dense_col` очищается
   для следующей итерации.

## 5. Детали реализации

```cpp
const auto &first = GetInput().first;
  const auto &second = GetInput().second;
  auto &result = GetOutput();

  std::vector<std::complex<double>> dense_col(first.count_rows, {0.0, 0.0});
  std::vector<int> active_rows;
  std::vector<bool> is_active(first.count_rows, false);

  for (int j = 0; j < second.count_cols; ++j) {
    for (int kb = second.col_index[j]; kb < second.col_index[j + 1]; ++kb) {
      int k = second.row_index[kb];
      std::complex<double> b_val = second.values[kb];

      for (int ka = first.col_index[k]; ka < first.col_index[k + 1]; ++ka) {
        int i = first.row_index[ka];
        if (!is_active[i]) {
          is_active[i] = true;
          active_rows.push_back(i);
        }
        dense_col[i] += first.values[ka] * b_val;
      }
    }

    std::ranges::sort(active_rows);

    for (int i : active_rows) {
      if (std::abs(dense_col[i]) > 1e-15) {
        result.values.push_back(dense_col[i]);
        result.row_index.push_back(i);
      }
      dense_col[i] = {0.0, 0.0};
      is_active[i] = false;
    }
    result.col_index[j + 1] = static_cast<int>(result.values.size());
    active_rows.clear();
  }

  return true;
```

## 6. Проверка корректности

Реализация успешно проходит модульное тестирование (GTest).
В рамках функциональных тестов проверяются краевые случаи:

1. `identity_2x2` — умножение на единичную матрицу.
2. `complex_scalar` — операции с комплексными числами.
3. `rectangular_simple` — прямоугольные матрицы (разные размеры).
4. `zero_matrix` — матрицы, состоящие из нулей.
5. `only_imaginary` — матрицы, содержащие только мнимую часть.
Тесты считывают эталонные матрицы из `.txt` файлов, вычисляют плотное (dense) умножение
как референс и сравнивают с результатом работы разреженного алгоритма CCS.

**Вывод:** Последовательная реализация полностью работоспособна, математически корректна

расходы. Данный код является отличным бейзлайном для распараллеливания.
