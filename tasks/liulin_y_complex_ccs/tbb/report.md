## 1. Контекст

Реализация на базе TBB ориентирована на использование паттерна Task-Based параллелизма.
В отличие от OMP, TBB предоставляет более гибкий планировщик задач (Work Stealing),
который эффективнее справляется с неравномерной вычислительной нагрузкой,
характерной для разреженных структур данных. Для оптимизации вычислений в этой версии
была изменена математическая схема: от линейной комбинации столбцов мы перешли
к скалярному произведению с предварительным транспонированием.

## 2. Постановка задачи

- **Вход/Выход:** Аналогично SEQ (две матрицы CCS `A` и `B`, результат `C = A * B`).
- **Ограничения:** Добавлена строгая проверка корректности формата CCS (метод `IsValidCCS`).
- **Цель:** Использовать возможности `tbb::parallel_for` для эффективного распределения
  нагрузки по ядрам процессора.

## 3. Схема алгоритма

Для того чтобы сделать внутренний цикл максимально быстрым и избежать использования
тяжеловесных векторов-аккумуляторов (как в SEQ/OMP), алгоритм был изменен:

1. **Транспонирование:** Матрица `A` ($M \times K$) транспонируется в `mat_at` ($K \times M$).
   Теперь строки исходной матрицы `A` доступны как столбцы `mat_at`.
2. **Параллельный перебор:** С помощью `tbb::parallel_for` мы распределяем обработку
   столбцов матрицы `B` (всего $N$ задач).
3. **Скалярное произведение:** Умножение строки $i$ матрицы `A` на столбец $j$ матрицы `B`
   превращается в пересечение двух разреженных векторов (столбец $i$ в `mat_at`
   и столбец $j$ в `mat_b`).
4. **Метод двух указателей:** Так как индексы строк в CCS упорядочены, скалярное произведение
   считается за один проход по двум столбцам (сложность $O(nnz(A_i) + nnz(B_j))$).

## 4. Особенности реализации

### Управление задачами

Используется `tbb::blocked_range<int>`, который позволяет библиотеке самостоятельно
определять оптимальный размер "порции" (grain size) столбцов для каждого ядра.

### Безопасность и исключения

Для корректной работы в многопоточной среде используется:

- `std::atomic<bool> success`: флаг для отслеживания ошибок во время выполнения
  параллельного цикла.
- Блок `try-catch` внутри `RunImpl`: если один из потоков столкнется с ошибкой
  (например, `std::bad_alloc`), выполнение будет прервано безопасно.

### Изоляция данных и Move-семантика

Чтобы избежать блокировок (mutex), каждый столбец $j$ имеет свои собственные векторы
для хранения результатов:

```cpp
std::vector<std::vector<std::complex<double>>> thread_values(res_cols);
std::vector<std::vector<int>> thread_row_indices(res_cols);
```

При финальной сборке используются **move-итераторы** (`std::make_move_iterator`),
что позволяет избежать лишнего копирования данных при объединении локальных результатов
в итоговую матрицу CCS.

## 5. Детали кода

```cpp
const auto &mat_a = GetInput().first;
  const auto &mat_b = GetInput().second;
  auto &mat_res = GetOutput();

  const int res_rows = mat_a.count_rows;
  const int res_cols = mat_b.count_cols;

  CCSMatrix mat_at;
  TransposeCCS(mat_a, mat_at);

  std::vector<std::vector<std::complex<double>>> thread_values(static_cast<size_t>(res_cols));
  std::vector<std::vector<int>> thread_row_indices(static_cast<size_t>(res_cols));
  std::atomic<bool> success{true};

  try {
    tbb::parallel_for(tbb::blocked_range<int>(0, res_cols), [&](const tbb::blocked_range<int> &r) {
      for (int j = r.begin(); j != r.end(); ++j) {
        ProcessColumn(j, res_rows, mat_at, mat_b, thread_values[static_cast<size_t>(j)],
                      thread_row_indices[static_cast<size_t>(j)]);
      }
    });
  } catch (...) {
    success = false;
  }

  if (!success) {
    return false;
  }

  mat_res.count_rows = res_rows;
  mat_res.count_cols = res_cols;
  mat_res.values.clear();
  mat_res.row_index.clear();
  mat_res.col_index.assign(static_cast<size_t>(res_cols) + 1, 0);

  for (int j = 0; j < res_cols; ++j) {
    const auto u_j = static_cast<size_t>(j);
    mat_res.values.insert(mat_res.values.end(), std::make_move_iterator(thread_values[u_j].begin()),
                          std::make_move_iterator(thread_values[u_j].end()));
    mat_res.row_index.insert(mat_res.row_index.end(), std::make_move_iterator(thread_row_indices[u_j].begin()),
                             std::make_move_iterator(thread_row_indices[u_j].end()));
    mat_res.col_index[u_j + 1] = static_cast<int>(mat_res.values.size());
  }

  return true;
```

## 6. Проверка корректности

TBB-версия прошла все функциональные тесты. Благодаря тому, что каждый поток работает

ненулевых элементов.
